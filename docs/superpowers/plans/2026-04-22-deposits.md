# Deposits Feature Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers-extended-cc:subagent-driven-development (recommended) or superpowers-extended-cc:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add optional per-event deposit configuration so admins can set a fixed deposit amount customers pay upfront, with the remainder due at the event.

**Architecture:** Extend the `Event` and `FirebaseEventsDocument` types with an optional `depositPrice` field (absent = no deposit, value = deposit enabled). Server actions handle tax calculation and FieldValue.delete() for removals. UI forms add a toggle + amount input following the existing sales-tax checkbox pattern.

**Tech Stack:** Next.js 15 App Router, React 19, TypeScript, Firebase Admin SDK (Firestore), shadcn/ui, Tailwind CSS v4, Sonner toasts.
